# 《诗岛》前端设计文档 V3

## 1. 产品定位

《诗岛》是一个基于诗人风格控制的七言绝句生成展示系统。用户在诗岛地图中进入不同诗人的场景，递出首句或心愿字条，同时对比**通用底座**和**风格模型**的生成结果。

当前版本支持两种条件生成：

```text
首句续写：给定第一句（7 字），生成后 3 句
藏头诗：给定 4 个字，依次作为四句首字
```

核心展示逻辑：

```text
诗岛地图 → 诗人场景 → 输入条件 → 双卷轴对比 → 点卷轴看手稿
```

|---|---|---|---|

```

│ 场景名称                                      │
```

### 双卷轴

| 位置 | 标签 | 后端路径 | 固定副标题 |
|---|---|---|---|
| 左 | **通用底座** | Transformer-6L + 规则约束 + 候选重排 | Transformer-6L · 规则约束 · 候选重排 |
| 右 | **风格模型** | Transformer + RAG上下文 + Poet Adapter | Transformer · RAG · Adapter |

- 两个卷轴在同一行并排显示
- 每个卷轴展示该模型的四句七言绝句
- 卷轴可点击：点哪个，弹出哪个的手稿面板
- 当前选中卷轴有绿色边框高亮

## 4. 诗人状态

进入场景时诗人先说一句话：

| 诗人 | 台词 |
|---|---|
| 王维 | 今日山雨将至，倒适合写一句清静诗。 |
| 李白 | 今夜月色正好，若无酒，也该有诗。 |
| 白居易 | 桥边听得几句闲谈，倒值得记下来。 |

角色有呼吸浮动动画和随机眨眼。

---

## 5. 条件生成模式

### 首句续写（"递一句诗"）

- 输入 7 个汉字
- 默认占位：`空山新雨晚风微`
- 第一句固定等于用户输入

### 藏头诗（"心愿笺"）

- 输入 4 个汉字
- 默认占位：`诗岛日常`
- 四句首字依次等于输入

---

## 6. 创作过程动画

不显示 loading 文字，改为诗人在创作：

```text
翻阅旧稿... → 推敲字句... → 誊写成诗...
```

每步间隔约 0.5 秒，结束后诗人恢复默认台词。

---

## 7. 手稿面板

- **不主动弹出**：生成完成后两个卷轴展示最佳作品，手稿面板默认关闭
- **点击卷轴打开**：点击左侧卷轴 → 弹出"通用底座 手稿"；点击右侧卷轴 → 弹出"风格模型 手稿"
- 面板从右滑入，包含 N 份候选（由"手稿数量"参数控制）
- 每份手稿显示：
  - 手稿编号 + 是否最佳
  - 四句七绝
  - 格式标签 / 主题一致性 / 风格分 / 重复度
  - 一句自然语言简评

示例：

```text
格式：通过
主题一致性：93%
风格分：88%
重复：低
简评：主题衔接表现好，没有明显跑题。风格辨识度也比较突出。
```

## 8. 参数设置

右上角齿轮打开创作偏好面板。参数同时影响两个卷轴。

| 参数 | 范围 | 默认 |
|---|---|---|
| temperature | 0.6 - 1.2 | 0.9 |
| top-k | 20 - 100 | 50 |
| 手稿数量 | 3 - 10 | 5 |

三个预设：

| 预设 | temperature | top-k | 手稿 |
|---|---|---|---|
| 稳定 | 0.7 | 30 | 3 |
| 平衡 | 0.9 | 50 | 5 |
| 灵感 | 1.2 | 100 | 8 |

滑杆移动时实时显示数值。

## 9. 前端接口

`POST /api/generate`：

请求：
```json
{
  "scene": "libai",
  "mode": "continue",
  "input": "春风又到小楼前",
  "temperature": 0.9,
  "topK": 50,
  "draftCount": 5
}
```

响应：
```json
{
  "ok": true,
  "general": {
    "title": "首句续写",
    "meta": "李白 · 通用底座 · T=0.9 top-k=50",
    "lines": ["春风又到小楼前", "...", "...", "..."]
  },
  "style": {
    "title": "首句续写",
    "meta": "李白 · RAG上下文 + Adapter · T=0.9 top-k=50",
    "lines": ["春风又到小楼前", "...", "...", "..."]
  }
}
```

API 不可用时前端本地兜底：通用底座和风格模型使用不同词库生成差异化结果。
